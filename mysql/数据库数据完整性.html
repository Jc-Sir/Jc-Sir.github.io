# 数据库数据完整性

指数据库中数据的准确性、一致性和可靠性。数据库数据完整性通常包括以下几个方面：

# 实体完整性

保证表中每行数据的唯一性和完整性，确保每个记录都具有唯一的主键，并且不能为空。

# 主键约束

  • 表中一列或者几列组合的值能用来唯一标识表中的每一行，这样的列或者列组合称为表的主键，主键表的数据不同重复。
  • 如果两列或者多列组合起来唯一标识表中的每一行，则该主键又称为"组合键"

WARNING

  • 最少性 尽量选择单个键作为主键
  • 稳定性 ，由于主键是用来在两个表间建立联接的，所以不能经常更新，最好就不更新

# 外键

外键用于连接两个表中的列，这些列必须具有相同的数据类型。它们将一个表中的数据与另一个表中的数据关联起来，并且保证了关联数据的完整性和一致性。

# 唯一约束

与主键约束不同的是，唯一约束允许列具有空值，但列值必须唯一。

例如: 学生的身份证号可以设置为唯一约束

# 标识列

WARNING

  • 1、一个表中只能有一个标识列
  • 2、标识列的类型只能是数值型：int、float、double
  • 3、标识列可以通过set auto_increment= 值；设置步长，也可以通过手动插入设置起始值
  • 4、标识列一定是在主键上吗？不一定，但要求是一个 key
-- 修改表时设置标识列
ALTER TABLE table_name MODIFY COLUMN table_column int PRIMARY KEY auto_increment;

-- 修改表时删除标识列
ALTER TABLE table_name MODIFY COLUMN table_column int;

# 外键约束

一个表的外键必须引用另一个表的主键,比如成绩表中的学生id会引用学生表的主键，课程id会引用成绩表的主键。

WARNING

  • 主表没有记录，子表中不能添加相应的记录
  • 修改和删除主表记录不能让子表记录孤立，必须相应修改和删除

# 参照完整性

保证表之间的关系的完整性，确保每个表中的外键都有一个对应的主键，并且不能引用不存在的记录。

# 域完整性

保证每个字段中存储的数据类型、长度、格式等符合定义，避免数据类型不匹配、数据溢出等错误。

# 用户定义完整性

根据特定需求，用户可以自定义数据验证规则，确保数据的完整性。

TIP

数据库数据完整性的保护可以采用多种技术和方法，如数据约束、触发器、存储过程、视图等。 通过这些方法，可以限制数据的输入、修改和删除操作，防止不合法的数据进入数据库，确保数据的正确性和可靠性。
